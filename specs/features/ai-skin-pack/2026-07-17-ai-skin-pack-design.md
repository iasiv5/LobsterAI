# AI 皮肤包功能设计文档

## 1. 概述

LobsterAI 已有颜色主题系统，但不能根据用户描述生成并应用整套视觉外观。AI 皮肤包把现有生图能力包装成一个受控工作流：用户通过内置外观定制 Kit 描述风格，系统串行生成背景和首页标记，注册到本地皮肤库并立即应用。

本功能不改变首页布局，也不允许模型注入 CSS 或重排组件。视觉表现由 LobsterAI 使用结构化配色和固定样式统一承载，以兼顾展示效果、可读性和默认主题兼容性。

### 1.1 目标

- 为订阅生图权益和用户自定义图片 Provider 提供统一的外观定制场景。
- 用可信 `skin_pack` 标记启动专用流程，不从普通对话文本猜测。
- 默认以约两次生图完成 `workspace.backdrop` 和 `home.emblem`，严格串行；异常恢复可追加尝试。
- 将结果复制到受管目录，支持预览、应用、停用、重新应用和删除。
- 让背景、配色和标记覆盖主要工作界面，同时复用现有主题系统处理明暗模式和可读性。

### 1.2 非目标

- 不生成 Icon 精灵图，不替换系统功能、文件类型、用户或 Agent 图标。
- 不允许任意 CSS、HTML、SVG、字体、动画、透明度或布局配置。
- 不提供导入导出、云同步、重命名或逐槽编辑。
- 不保证跨进程恢复未完成的生图任务，也不自动重试付费请求。

## 2. 核心流程

1. 用户选择内置外观定制 Kit 并输入主题、氛围和配色描述。
2. 运行时确认可信 Kit 的 `skin_pack` 标记并创建草稿。
3. 已登录且具有图片权益时使用 `lobsterai_image_generate`；否则在 OpenClaw 原生 `image_generate` 可用时使用用户配置的 Provider。
4. 先生成并注册 `workspace.backdrop`，成功后再生成 `home.emblem`；同一皮肤锁定相同后端和模型。
5. 两个槽位均通过校验后写入受管皮肤库并应用。
6. 用户可在“设置 - 外观”中恢复默认、重新应用或删除本地皮肤，以上操作均不再次生图。

没有可用生图能力时，流程在请求生成前停止，并提示用户登录订阅或配置 Provider。

## 3. 功能要求

### 3.1 工作流与工具路由

- Kit 使用稳定 ID 和结构化 `skin_pack` 元数据；非皮肤会话不能调用皮肤管理工具。
- 两个图片槽位是首版唯一合法槽位。约两次生成是 Skill 指令中的软预算，不在主进程设置不可扩展的硬次数上限。
- 所有生图调用串行执行；失败后只允许在已锁定的后端内恢复，不自动跨后端回退。
- LobsterAI 异步图片任务由工具内部轮询至成功、失败、取消或超时，避免模型并发查询。

### 3.2 资产注册与安全

- 注册只接受生图工具返回的精确本地路径或 `file://` URL，不扫描工作目录猜测产物。
- 仅允许可解码的 PNG、JPEG、WebP，并校验文件大小和图片尺寸。
- 资产复制到 `userData/skins/<skinId>/assets`，使用内容哈希文件名；manifest 只保存受管相对路径。
- `lobster-skin://` 协议只能读取注册表中已登记且仍位于受管目录内的资产。
- 两个必需槽位均有效时才可应用；半成品保留为草稿但不影响当前皮肤。

### 3.3 结构化表现层

完整皮肤可携带宿主校验的 `immersive_shell` 表现配置：

- 语义色：`canvas`、`panel`、`panelRaised`、`accent`、`accentForeground`、`accentAlt`、`foreground`、`muted`、`border`。
- 构图焦点：`focusX`、`focusY`。
- 可选粒子密度。

宿主校验颜色格式和前景对比度，并由固定 CSS 映射到下列区域：

- Cowork 首页和会话背景、首页标记、快捷操作和输入区。
- Sidebar、应用标题栏和会话标题栏。
- 专家套件、技能、MCP、定时任务的页面画布。

管理页中的卡片、表单、搜索框、代码和凭据区域保持实色背景；设置、登录、欢迎页和 Portal 弹窗不纳入皮肤表现层。背景透明度、遮罩强度和组件布局全部由 LobsterAI 控制，manifest 不能覆盖。

### 3.4 明暗模式兼容

- `preferredAppearance` 根据结构化画布、面板和前景色自动推断为 `light` 或 `dark`，不接受与配色矛盾的声明。
- 应用新皮肤时复用现有 `ThemeService`：若当前主题已经是目标明暗模式，保留其具体主题；否则切换到同明暗模式的兼容主题。
- 同一皮肤激活只自动适配一次，之后用户手动选择主题应优先。
- 停用或删除皮肤不自动恢复先前主题，用户可继续使用当前颜色主题。
- 默认主题、纯颜色主题以及缺少结构化表现配置的旧皮肤不触发上述适配。

### 3.5 本地皮肤管理

- 外观设置列出所有 `ready` 皮肤，使用真实背景和首页标记预览。
- 非当前皮肤可直接应用；恢复默认只停用图片皮肤并保留本地资产。
- 删除前必须确认。删除当前皮肤时先清除 active 指针，再删除对应 manifest 和受管资产并通知 Renderer。
- 删除入口只通过可信 Electron IPC 暴露给设置页，不开放给模型工具。
- 不删除会话目录中的原始生图文件，也不影响 Kit、Skill 或其他皮肤。

## 4. 实现边界

| 层级 | 职责 |
|------|------|
| `SKILLs/skin-creator/` | 主题理解、两槽位提示词、串行顺序、工具选择和停止条件 |
| `src/shared/skin/` | 工作流、槽位、表现配置、IPC 和协议类型 |
| `src/main/skins/` | 会话事务、媒体桥、资产校验、存储、协议、IPC 和 Kit 生命周期 |
| `openclaw-extensions/lobster-media-generation/` | 会员生图状态轮询和本地皮肤管理工具桥 |
| `src/renderer/components/skin/` | 表现作用域、背景、皮肤库与管理交互 |
| `src/renderer/services/skin*.ts` | Renderer 状态同步和明暗模式适配 |

主进程按上述领域模块维护，`main.ts` 只负责组合和委托；Renderer 通过 preload API 获取规范化 manifest，不接触任意本地路径。

## 5. 边界情况

| 场景 | 处理方式 |
|------|---------|
| 用户要求额外图片或图标 | 忽略首版槽位之外的要求 |
| 生图失败、超时或没有本地结果 | 保持串行并允许同后端恢复；无法恢复则停止 |
| 应用时缺少槽位 | 拒绝应用，原 active skin 保持不变 |
| 背景加载失败 | 保留语义色和实色内容表面 |
| 旧皮肤没有 `presentation` | 继续使用原颜色主题和基础图片行为 |
| 应用运行中退出 | 不恢复未完成草稿；已完成 active skin 可在启动时加载 |
| 删除失败 | 注册表保持一致，并向设置页返回错误 |

## 6. 验收标准

1. 用户可从内置 Kit 输入任意明确风格，按确定路线串行生成并应用两个必需资产。
2. 受管 manifest 不引用会话工作目录，非 `skin_pack` 会话不能注册皮肤。
3. 首页、会话、Sidebar、标题栏及四类管理页呈现统一风格，核心内容仍清晰可读。
4. 皮肤自动采用合适的明暗模式，同时复用现有主题服务，不污染默认和纯颜色主题。
5. 设置页可预览、应用、停用和删除本地皮肤；重新应用不触发生图。
6. 切换 Agent、会话和管理页时不重复挂载同一背景图片，不出现明显背景或遮罩闪烁。
