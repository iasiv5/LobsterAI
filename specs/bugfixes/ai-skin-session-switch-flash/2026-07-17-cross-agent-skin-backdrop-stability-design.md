# 跨 Agent 切换会话时 AI 皮肤遮罩闪烁修复设计

## 1. 概述

### 1.1 问题

启用 AI 皮肤后，在同一 Agent 内切换会话时背景表现稳定；从一个 Agent
的会话切换到另一个 Agent 的已有会话时，背景图片本身不会重新加载，但深色
皮肤会短暂变亮，随后恢复为会话页的较暗遮罩。

### 1.2 根因

跨 Agent 会话导航会先切换 Agent 并清空 `currentSession`，再加载目标 Agent
的会话列表和目标会话。Cowork 页面此前仅根据 `currentSession` 是否存在选择
首页或会话背景：

- `currentSession` 为空时使用首页背景表现；
- `currentSession` 存在时使用会话背景表现。

因此异步加载期间会短暂进入首页背景表现。首页与会话页使用不同的图片透明度
和遮罩强度，最终形成“先亮后暗”的视觉闪烁。

## 2. 用户场景

### 场景 1：跨 Agent 打开已有会话

**Given** 用户已经启用 AI 皮肤并正在查看一个会话
**When** 用户打开另一个 Agent 下的已有会话
**Then** 背景应始终保持会话页遮罩，不应短暂切换为首页遮罩或首页粒子效果

### 场景 2：切换 Agent 后新建任务

**Given** 用户正在查看一个会话
**When** 用户切换 Agent 并新建任务
**Then** 页面应正常进入首页背景表现，不应错误保留会话遮罩

### 场景 3：目标会话加载失败

**Given** 页面正在跨 Agent 加载已有会话
**When** 会话列表或目标会话加载失败
**Then** 临时会话展示状态必须释放，页面可以回落到正常首页表现

## 3. 功能需求

### FR-1：展示状态与数据状态分离

新增仅用于跨 Agent 会话导航的 `sessionNavigationTargetId`。该字段不得替代
`currentSessionId`，也不得被消息发送、停止任务、通知抑制、草稿或消息流逻辑
当作已经加载完成的当前会话。

### FR-2：稳定 AI 皮肤表现

当 `currentSession` 存在，或 `sessionNavigationTargetId` 存在时，Cowork 页面
都应使用会话背景表现并隐藏首页粒子效果。

### FR-3：正确完成和回落

- 目标会话加载成功后清除临时目标；
- 加载失败或抛出异常后清除匹配的临时目标；
- 只允许当前目标完成操作清除状态，旧请求不得清除更新的目标；
- 普通 Agent 切换、新建任务和显式返回首页应清除临时目标。

## 4. 实现方案

1. 在 Cowork Redux 状态增加 `sessionNavigationTargetId`。
2. `clearCurrentSession` 接受可选的目标会话 ID，使 Agent 和会话展示意图能够
   在同一次 Redux 更新中完成，避免出现中间渲染帧。
3. `agentService.switchAgent` 接受可选 `targetSessionId`：
   - 打开跨 Agent 已有会话时传入目标 ID；
   - 普通 Agent 切换和新建任务不传入。
4. 侧栏任务、任务搜索结果和 Agent 任务快捷入口在完整异步导航结束后释放
   匹配的临时目标。
5. `CoworkView` 使用“当前会话或临时目标存在”判断背景表现；实际会话内容仍
   只依据 `currentSession` 渲染。

## 5. 边界情况

| 场景 | 处理方式 |
|------|----------|
| 同一 Agent 切换会话 | 沿用既有流程，当前会话在目标返回前仍存在 |
| 快速连续选择不同会话 | 完成操作携带目标 ID，仅清除匹配目标 |
| 会话 IPC 不可用或抛错 | `finally` 释放匹配目标 |
| 切换 Agent 但不打开会话 | 不设置临时目标，正常显示首页 |
| 删除会话或点击新建任务 | 显式清空当前会话时同步清除临时目标 |
| 未启用 AI 皮肤 | 没有背景资产和粒子效果，原主题色表现不变 |

## 6. 验收标准

1. 深色 AI 皮肤下跨 Agent 切换已有会话时，不再出现背景先亮后暗。
2. 同一 Agent 内切换会话的既有稳定表现不退化。
3. 新建任务仍显示首页背景强度和首页粒子效果。
4. 加载失败后不会永久停留在会话遮罩状态。
5. 默认主题和纯颜色主题的页面配色、消息和 Agent 行为不变。
6. Cowork reducer、Agent 切换和清空会话相关测试通过。
