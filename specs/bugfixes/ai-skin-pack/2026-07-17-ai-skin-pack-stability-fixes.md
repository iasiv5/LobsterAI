# AI 皮肤切换稳定性修复汇总

## 1. 问题与根因

AI 皮肤首版在会话和 Agent 切换时出现过两类短暂闪烁：

- 背景节点随页面内容重新挂载，已加载图片会再次进入加载状态。
- 跨 Agent 打开已有会话时，`currentSession` 会在异步加载期间短暂为空，页面误用首页遮罩，深色皮肤表现为先亮后暗。

这些问题只影响展示稳定性，不涉及皮肤数据、消息内容或 Agent 路由。

## 2. 修复方案

- 将背景放到稳定的皮肤表现容器中，预加载资产并在首页、会话和管理页之间复用。
- 增加仅用于展示判断的 `sessionNavigationTargetId`。跨 Agent 打开已有会话时，即使实际会话尚未加载完成，也继续使用会话遮罩。
- 目标加载完成、失败、返回首页或新建任务时清理临时目标；旧请求不能清除更新的目标。
- 实际消息、发送、停止、草稿等逻辑仍只读取 `currentSession`，不把展示状态当作业务状态。

## 3. 兼容性与验收

- 同一 Agent 切换会话保持原流程。
- 普通 Agent 切换和新建任务仍进入首页表现。
- 未启用 AI 皮肤时没有新增可见效果。
- 深色和浅色皮肤跨 Agent 打开已有会话时，背景与遮罩均不应出现明显闪烁。
- reducer、Agent 切换和会话清理测试覆盖临时目标的建立、匹配清除和异常回落。
